# ❓ 常见问题

解决使用过程中遇到的常见问题

## 🚀 快速开始问题

### Q: 无法连接多人游戏服务器怎么办？

**A:** 连接问题可能有以下几种情况：

1. **网络连接问题**
   - 检查网络连接是否正常
   - 尝试刷新页面重新连接
   - 切换网络环境（如切换到手机热点）

2. **服务器暂时不可用**
   - 等待1-30秒，服务器可能正在启动
   - 可以先使用单人模式体验功能
   - 稍后再尝试多人游戏

3. **浏览器兼容性**
   - 建议使用Chrome、Firefox、Edge等现代浏览器
   - 避免使用过旧版本的浏览器
   - 清除浏览器缓存后重试

### Q: 加入房间时提示"房间不存在"？

**A:** 请检查以下几点：

- **房间代码准确性**：确保输入的4位数字代码正确
- **房间状态**：主持人可能已经离开，导致房间关闭
- **网络延迟**：稍等片刻再尝试加入
- **重新创建**：如果确认代码正确但仍无法加入，请主持人重新创建房间

### Q: 为什么点击"进去看看"没有反应？

**A:** 可能的原因和解决方案：

- **JavaScript未加载**：刷新页面确保所有脚本正常加载
- **浏览器限制**：检查是否禁用了JavaScript
- **网络问题**：确保网络连接稳定
- **缓存问题**：清除浏览器缓存后重试

## 🎭 角色抽取问题

### Q: 抽取角色时显示空白或错误？

**A:** 解决步骤：

1. **检查角色启用状态**
   - 进入设置 → 角色管理
   - 确保至少启用了4个角色
   - 检查筛选条件是否过于严格

2. **清除浏览器数据**
   ```
   按F12 → Application → Storage → Clear site data
   ```

3. **重置游戏状态**
   - 设置 → 游戏设置 → 重置游戏
   - 确认重置后重新开始

### Q: BP模式工作异常，重复抽到相同角色？

**A:** 检查BP模式配置：

- **全局BP模式**：确认没有手动重置BP状态
- **个人BP模式**：每个玩家独立计算，允许不同玩家获得相同角色
- **关闭BP模式**：完全随机，可能出现重复角色
- **屏蔽机制**：被屏蔽的角色不会再次出现

### Q: 点击角色卡片重抽没有效果？

**A:** 可能的原因：

1. **可用角色不足**
   - 检查剩余可用角色数量
   - 在角色管理中启用更多角色
   - 考虑调整BP模式设置

2. **网络同步问题**（多人游戏）
   - 等待网络同步完成
   - 刷新页面重新加入房间

3. **功能冲突**
   - 避免在抽取过程中频繁点击
   - 等待动画完成后再进行操作

## 📋 事件系统问题

### Q: 抽取事件时没有显示任何内容？

**A:** 排查步骤：

1. **检查事件启用状态**
   - 设置 → 事件管理
   - 确保个人事件和团队事件中至少有事件被启用

2. **检查游戏设置**
   - 设置 → 游戏设置
   - 确认"启用个人事件抽取"和"启用团队事件抽取"开关状态

3. **数据加载问题**
   - 刷新页面重新加载事件数据
   - 检查浏览器控制台是否有错误信息

### Q: 团队事件投票界面不显示？

**A:** 可能的原因和解决方案：

- **团队事件开关关闭**：检查游戏设置中的团队事件抽取开关
- **没有启用团队事件**：在事件管理中启用至少3个团队事件
- **多人游戏同步问题**：确保所有玩家都在同一房间
- **浏览器兼容性**：使用支持现代Web标准的浏览器

### Q: 重抽次数显示异常或无法调整？

**A:** 解决方法：

1. **手动调整**
   - 使用重抽次数旁的+/-按钮调整
   - 确保数值在合理范围内（通常0-10次）

2. **重置重抽次数**
   ```javascript
   // 在浏览器控制台执行
   localStorage.removeItem('rerollCount');
   location.reload();
   ```

3. **检查事件设置**
   - 确认个人事件抽取开关已开启
   - 重抽次数只在个人事件启用时显示

## ⚙️ 设置和配置问题

### Q: 设置无法保存或丢失？

**A:** 解决方案：

1. **浏览器存储问题**
   - 检查浏览器是否允许本地存储
   - 清除过期的localStorage数据
   - 在隐私模式下可能无法保存设置

2. **存储空间不足**
   ```javascript
   // 检查localStorage使用情况
   console.log(JSON.stringify(localStorage).length);
   ```

3. **手动备份设置**
   - 记录重要的角色和事件配置
   - 截图保存设置状态以便恢复

### Q: 角色筛选功能不工作？

**A:** 检查筛选配置：

1. **筛选条件冲突**
   - 检查是否选择了矛盾的筛选条件
   - 清除所有筛选条件后重新设置

2. **搜索关键词**
   - 确认搜索框中没有残留的关键词
   - 尝试清空搜索框

3. **数据加载**
   - 确保角色数据已完全加载
   - 刷新页面重新初始化筛选功能

### Q: 移动端操作困难？

**A:** 移动端优化建议：

1. **界面适配**
   - 使用横屏模式获得更好的体验
   - 放大页面以便更精确的操作

2. **网络连接**
   - 确保移动网络稳定
   - 建议在WiFi环境下使用

3. **功能限制**
   - 推荐移动端作为参与者而非主持人
   - 复杂设置建议在PC端完成

## 🌐 多人游戏问题

### Q: 多人游戏中状态不同步？

**A:** 同步问题解决：

1. **网络连接检查**
   - 确保所有玩家网络连接稳定
   - 主持人网络连接尤为重要

2. **强制同步**
   - 所有玩家刷新页面重新加入
   - 主持人重新创建房间

3. **浏览器缓存**
   - 清除浏览器缓存
   - 使用隐私模式测试

### Q: 主持人离开后游戏无法继续？

**A:** 这是正常的设计行为：

- **房间生命周期**：主持人离开时房间自动关闭
- **应对方案**：选择网络稳定的玩家作为主持人
- **备用方案**：准备好随时重新创建房间的方案

### Q: 投票结果显示错误？

**A:** 投票系统问题排查：

1. **同步延迟**
   - 等待几秒钟让投票结果完全同步
   - 避免快速连续投票

2. **重新投票**
   - 清除当前投票状态
   - 重新进行团队事件抽取

3. **网络问题**
   - 检查所有玩家的网络连接
   - 必要时重新加入房间

## 🔧 技术问题

### Q: 浏览器控制台出现错误信息？

**A:** 常见错误处理：

1. **JavaScript错误**
   ```
   Uncaught ReferenceError: xxx is not defined
   ```
   - 刷新页面重新加载脚本
   - 清除浏览器缓存
   - 检查网络连接

2. **网络错误**
   ```
   Failed to load resource
   ```
   - 检查网络连接
   - 确认服务器状态
   - 尝试使用其他网络

3. **存储错误**
   ```
   QuotaExceededError
   ```
   - 清除localStorage数据
   - 重置游戏状态

### Q: 页面加载缓慢或卡顿？

**A:** 性能优化：

1. **浏览器优化**
   - 关闭不必要的浏览器标签页
   - 清除浏览器缓存和Cookie
   - 更新浏览器到最新版本

2. **网络优化**
   - 使用有线网络替代WiFi
   - 关闭其他占用带宽的应用
   - 选择网络状况良好的时间段

3. **设备性能**
   - 关闭其他占用内存的程序
   - 确保设备有足够的可用内存
   - 考虑使用性能更好的设备

## 🆘 紧急解决方案

### 完全重置解决方案

当遇到严重问题时，可以尝试完全重置：

```javascript
// 在浏览器控制台执行以下代码
localStorage.clear();
sessionStorage.clear();
location.reload();
```

### 数据恢复

如果重要数据丢失：

1. **检查浏览器历史**：某些浏览器可能保留了历史数据
2. **重新配置**：根据使用习惯重新设置角色和事件
3. **备份习惯**：建议定期截图保存重要配置

### 联系技术支持

如果以上方法都无法解决问题：

1. **收集错误信息**：记录具体的错误提示和操作步骤
2. **环境信息**：提供浏览器版本、操作系统等信息
3. **重现步骤**：详细描述问题出现的具体操作流程

---

**大部分问题都可以通过上述方法解决。如果问题持续存在，建议尝试使用不同的浏览器或设备进行测试。** 🛠️
